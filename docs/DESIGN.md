# Design Spec (Gate 3b)

## 문서 목적
이 문서는 Gate 3a(`docs/DESIGN_RESEARCH.md`)에서 추출된 규칙을 구현 가능한 디자인 스펙으로 확정하는 템플릿이다.
모든 항목은 Gate 5 구현, Gate 6 QA, Gate 7 컴플라이언스에서 추적 가능하도록 결정문으로 작성한다.

## 1) Design Mode 적용 규칙(A/B/C)
- 선택 모드:
- 선택 근거:

### Mode A: Minimal (감성/여백 중심)
- 결정: 한 화면의 핵심 행동은 최대 1개 CTA로 제한한다.
- 결정: 본문 정보 블록은 최대 3개 그룹으로 나눈다.
- 결정: 카드/섹션 간 여백은 Mode B 대비 +20% 적용한다.

### Mode B: Balanced (기본값)
- 결정: 정보 밀도와 가독성 균형을 유지하기 위해 기본 그리드와 간격 토큰을 표준값으로 사용한다.
- 결정: 액션 우선순위는 Primary 1개 + Secondary 1개까지 허용한다.
- 결정: 테이블/리스트는 핵심 컬럼 우선 노출, 상세는 확장 영역으로 분리한다.

### Mode C: Dense (유틸/생산성/어드민)
- 결정: 동일 뷰포트 내 표시 정보량을 Mode B 대비 20~30% 확대한다.
- 결정: 목록/테이블의 기본 행 높이는 Mode B 대비 축소하되 최소 터치 타겟 기준은 유지한다.
- 결정: 고급 필터/일괄 액션을 1 depth에서 접근 가능하게 배치한다.

## 2) 코어 디자인 원칙(Apple-like 기본)
1. 결정: 핵심 목적과 직접 관련 없는 장식 요소는 배치하지 않는다.
2. 결정: 타이포 위계를 통해 제목/본문/보조 정보의 우선순위를 시각적으로 고정한다.
3. 결정: 콘텐츠 그룹 간 여백 차이를 최소 2단계 이상 유지한다.
4. 결정: 동일한 사용자 의도에는 동일한 컴포넌트 패턴을 사용한다.
5. 결정: 포인트 컬러는 1개만 사용하고 의미를 혼합하지 않는다.
6. 결정: 모션은 이해 보조 목적에 한해 사용하고 장식 모션은 금지한다.
7. 결정: 인터랙션 결과(성공/오류/로딩)는 1초 이내 가시 피드백을 제공한다.
8. 결정: 명도 대비와 터치 타겟은 기본 접근성 기준 이하로 낮추지 않는다.
9. 결정: 민감 정보는 기본 마스킹 정책을 따른다.
10. 결정: 광고성/프로모션성 UI는 기능 UI와 명확히 구분한다.

## 3) 토큰 정의(숫자/규칙)

## 4) 컴포넌트 결정문
| 컴포넌트 | 상태 | 결정문 |
|---|---|---|
| 버튼 | 기본/포커스/비활성/로딩 | 결정: Primary는 화면당 핵심 CTA 1개로 제한한다. 로딩 중 중복 클릭을 차단한다. |
| 입력 | 기본/포커스/오류/비활성/빈값 | 결정: 라벨은 항상 노출한다. 오류 메시지는 색상 외 텍스트 설명을 포함한다. |
| 카드 | 기본/호버/선택/비활성 | 결정: 카드 전체 클릭 영역과 내부 보조 액션 영역을 분리한다. |
| 모달 | 기본/포커스/로딩/오류 | 결정: ESC 닫기와 포커스 트랩을 기본 적용한다. |
| 탭 | 기본/활성/비활성 | 결정: 키보드 좌우 이동과 활성 상태 대비를 필수로 제공한다. |
| 토스트 | 기본/표시/사라짐 | 결정: 자동 닫힘은 3~5초 범위로 통일하고, 치명 오류는 수동 닫기 허용한다. |

## 5) 상태(State) 규칙
- 결정: 기본 상태는 시각적 노이즈를 최소화한다.
- 결정: 포커스 상태는 색상 외 윤곽선/두께 변화로 식별 가능해야 한다.
- 결정: 비활성 상태는 기능 비활성 이유를 툴팁 또는 설명으로 제공한다.
- 결정: 로딩 상태는 300ms 이상 지연 시 스켈레톤 또는 진행 피드백을 제공한다.
- 결정: 빈값 상태는 다음 행동 CTA를 함께 제시한다.
- 결정: 오류 상태는 복구 행동(재시도/문의)을 명시한다.

## 6) 모션 규칙
- 결정: 빠른 피드백 모션은 120~180ms로 제한한다.
- 결정: 일반 전환은 200~240ms를 기본값으로 한다.
- 결정: 레이아웃 변화는 최대 320ms를 초과하지 않는다.
- 결정: 허용 모션은 페이드, 미세한 Y축 이동, 소규모 스케일로 제한한다.
- 결정: 금지 모션은 반복 장식 애니메이션, 과도한 바운스, 멀미 유발 패턴이다.

## 7) 반응형 규칙
- 결정: 모바일 우선 설계를 기본으로 하고, 브레이크포인트는 `>=768`, `>=1024`, `>=1280` 기준으로 확장한다.
- 결정: 주요 CTA는 모바일 첫 뷰포트 내에서 접근 가능해야 한다.
- 결정: Dense 모드에서도 터치 타겟 최소 44x44px를 유지한다.
- 결정: 2열 이상 레이아웃은 정보 우선순위를 유지하는 경우에만 적용한다.

## 8) 접근성/컴플라이언스 추적 훅
- 결정: 텍스트/아이콘 대비는 WCAG AA 이상을 기본값으로 유지한다.
- 결정: 키보드만으로 핵심 플로우를 완료할 수 있어야 한다.
- 결정: 폼 입력은 시각적 라벨과 프로그램적 라벨을 모두 제공한다.
- 결정: 개인정보/민감정보는 기본 마스킹 후 노출 권한을 분리한다.
- 결정: 광고성/추천성 UI는 표시광고 정책 검토 항목을 PR 체크리스트에 연결한다.

## 9) Gate 5/6/7 추적 포인트
- Gate 5 구현 체크 포인트:
  1. 토큰 값이 코드에 일치 적용되었는지
  2. 컴포넌트 상태 규칙이 모두 구현되었는지
- Gate 6 QA 체크 포인트:
  1. Mode별 레이아웃/정보 밀도 검증
  2. 오류/빈값/로딩 상태 회귀 검증
- Gate 7 컴플라이언스 체크 포인트:
  1. 접근성 기준 충족 여부
  2. 개인정보/표시광고 리스크 여부
