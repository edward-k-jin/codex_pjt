# OPS Lessons Learned 템플릿

## 사용 원칙
- 오너 피드백/수정이 발생하면 반드시 항목을 추가한다.
- 각 항목은 "문제 패턴 / 원인 / 해결 / 재발 방지 규칙" 4요소를 모두 채운다.
- 재발 방지 규칙은 다음 작업에서 즉시 적용 가능한 문장으로 쓴다.

## 기록 포맷
| 날짜 | Gate | 문제 패턴 | 원인 | 해결 | 재발 방지 규칙(다음부터 이렇게 한다) | 작성자 |
|---|---|---|---|---|---|---|
| 2026-02-24 | Gate 0 | 예: 병렬 트랙이 같은 파일을 동시에 수정함 | 파일 스코프 사전 합의 누락 | 트랙별 파일 소유권 재정의 | 다음부터 계획 단계에서 파일 소유권 표를 먼저 확정한다 | A0 |

## 누적 메모
- 예: QA 증거 링크 누락이 반복되어 PR 템플릿에 증거 항목을 고정했다.
- 예: 컴플라이언스 검토가 늦어지는 문제를 해결하기 위해 Gate 2부터 법무 리스크를 사전 표기한다.
