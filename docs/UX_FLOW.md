# UX 플로우 운영 템플릿

## 문서 목적
이 문서는 UX 에이전트가 정보 구조, 사용자 플로우, 화면 상태, 카피/접근성 기준을 채우기 위한 템플릿이다.

## 단일 진실의 원천(SSOT)
- PRD는 `docs/PRD.md`가 단일 진실의 원천이다.
- 이 문서의 모든 플로우/상태는 PRD의 범위와 수용 기준에 종속된다.
- PRD와 충돌 시 UX 문서를 수정하고, 필요하면 Gate 1 재승인을 받는다.

## 오너 승인
`[오너 승인] 게이트 2 승인 완료 - <오너 이름> - <연도-월-일>`

## 1) 정보 구조(IA)
| 레벨 | 화면/영역 | 목적 | 주요 콘텐츠 | 비고 |
|---|---|---|---|---|
| L1 | 예: 홈 | 서비스 진입과 핵심 행동 유도 | 주요 CTA, 최근 항목 | 비회원 진입 가능 |
| L2 | 예: 상세 | 항목 확인/편집 | 상세 정보, 보조 액션 | 로그인 업그레이드 포인트 표시 |

## 2) 주요 사용자 플로우

### 플로우 A: 게스트 빠른 시작
- 시작 조건: 첫 방문 사용자가 로그인 없이 접근한다.
- 단계 1: 홈에서 핵심 CTA를 선택한다.
- 단계 2: 입력/선택을 완료하고 결과를 확인한다.
- 단계 3: 임시 저장(localStorage 또는 IndexedDB) 상태를 확인한다.
- 완료 조건: 사용자 목표를 로그인 없이 달성한다.

### 플로우 B: 로그인 업그레이드
- 시작 조건: 게스트 사용자가 동기화/결제/협업 등 서버 기능이 필요하다.
- 단계 1: 업그레이드 안내 모달 또는 배너를 노출한다.
- 단계 2: 로그인/회원가입 후 인증을 완료한다.
- 단계 3: 게스트 데이터와 계정 데이터를 병합한다.
- 완료 조건: 서버 저장 상태에서 동일 작업을 이어서 수행한다.

## 3) 화면별 스펙
| 화면 ID | 화면명 | 목적 | 진입 조건 | 핵심 컴포넌트 | 사용자 액션 | 상태(로딩/빈값/오류/성공) | 카피 톤 | 접근성 포인트 |
|---|---|---|---|---|---|---|---|---|
| SCR-01 | 예: 홈 | 핵심 작업 시작 | 앱 최초 진입 | 헤더, CTA 버튼, 요약 카드 | 시작하기 클릭 | 로딩 스켈레톤/빈값 안내/오류 토스트/성공 배지 | 짧고 명확한 안내 톤 | 포커스 순서, 버튼 라벨 명확화 |
| SCR-02 | 예: 결과 | 결과 확인 및 다음 행동 | 플로우 A 단계 2 완료 | 결과 카드, 공유 버튼 | 저장/공유 선택 | 로딩 인디케이터/빈 결과 안내/재시도 메시지/성공 토스트 | 신뢰 중심 설명 톤 | 대비 기준, 키보드 조작 가능 |

## 4) 권한/알림/퍼미션(해당 시)
- 알림 권한 필요 여부: 예: 마감 알림 기능에서만 요청.
- 카메라/파일 권한 필요 여부: 예: 첨부 업로드 기능에서만 요청.
- 권한 거부 시 대체 동선: 예: 수동 입력 경로 제공.
- 알림 해제 동선: 예: 설정 > 알림에서 즉시 해제 가능.

## 5) UX 수용 기준 매핑(PRD 연계)
| PRD 수용 기준 ID | UX 반영 위치 | 검증 방법 | 메모 |
|---|---|---|---|
| AC-01 | 플로우 A 단계 1~3 | QA 시나리오 QA-AC-01 | 예: 비회원 3단계 이내 완료 |
| AC-02 | 플로우 B 단계 1~3 | QA 시나리오 QA-AC-02 | 예: 로그인 후 데이터 병합 성공 |

## 6) 미해결 이슈/질문(최대 5개)
1. 예: 게스트 데이터 보관 기간을 7일로 고정할지 결정 필요.
2. 예: 업그레이드 모달 노출 타이밍 AB 테스트 여부 확인 필요.
3. 예: 빈값 화면의 추천 콘텐츠 노출 정책 확인 필요.
4. 예: 접근성 문구(스크린리더 전용) 표준안 확정 필요.
5. 예: 모바일 우선 레이아웃 우선순위 확정 필요.
